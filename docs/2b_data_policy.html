9  Data Flow and Data Retention

Last updated: 2025-08-18

This document describes the data retention policy for the Public AI Data Flywheel integrated with OpenWebUI at https://app.publicai.company/.

9.1 How Users Create Data

9.1.1 Using OpenWebUI (No Sharing)

When you use OpenWebUI normally: - Chat data: Stored in OpenWebUI database for your history - Account data: Email, username, preferences - Not shared: This data is NOT sent to the flywheel unless you explicitly opt in

9.1.2 Sharing via the Flywheel Action

When you enable sharing and trigger the action: 1. One-time setup: Enable sharing in Settings / User Valves #todo better instructions here 2. Per-chat action: Explicitly trigger “Public AI Data Flywheel” 3. Confirmation required: Review settings and confirm each share 4. Data sent: Chat content, metadata, your chosen license/preferences

9.3 Data Processing Pipeline

9.3.1 1. User Contribution

  • User enables sharing in User Valves
  • User triggers flywheel action in chat
  • System shows license, AI preference, attribution
  • User confirms or cancels

9.3.2 2. Waiting Room

Location: waiting_room/{contributor_hash}{timestamp}.json Contains: Full contribution JSON Duration: Until daily processing script runs Access: Repository maintainers only

9.3.3 3. PII Redaction

Automated detection and removal of: - Email addresses - IP addresses (IPv4/IPv6) - Social Security Numbers - IBAN accounts - Crypto wallets (ETH/BTC) - Phone numbers - Credit cards (Luhn-validated) - Name patterns (“my name is X”)

9.3.4 4. Validation & Processing

  • Files with >1 PII hits → quarantine #todo: tune the threshold
  • Malformed JSON → quarantine
  • Valid files → append to monthly release file

9.3.5 5. Public Release

  • Format: JSONL files partitioned by month
  • Location: HuggingFace publicai/shared-chats
  • Access: Gated (request required)
  • License: As specified by contributor
  • Permanence: Cannot be retracted

9.4 User Rights & Controls

9.4.1 What You Control

  • Opt-in toggle: Must explicitly enable sharing
  • License choice: CC0, CC-BY, or CC-BY-SA
  • AI preferences: IETF/CC signals for downstream use
  • Attribution: Username, anonymous, or pseudonym
  • Per-chat consent: Confirm each contribution

9.4.2 What You Cannot Control

  • Published data: Once released, cannot be deleted
  • Downstream use: Subject to license terms, enforcement may vary!
  • Provider behavior: When chats sent to AI providers

9.4.3 Deletion Requests

  • Future releases: Can exclude your data going forward
  • Past releases: Generally permanent and irrevocable
  • Contact: privacy@publicai.company for requests

9.5 Contributor Identifiers

9.5.1 Contributor Hash

  • Format: 16-character hex string
  • Generation: SHA256(salt + "openwebui:" + user_id)[:16]
  • Purpose: Group anonymous contributions
  • Not stored: Raw user_id never saved

9.5.2 Attribution Options

  1. Use username: Your OpenWebUI username
  2. Anonymous: No attribution
  3. Custom pseudonym: Your chosen name

9.6 Security Measures

9.6.1 In Transit

  • HTTPS/TLS for all connections
  • API tokens for HuggingFace uploads

9.6.2 At Rest

  • HuggingFace repository access controls
  • No additional encryption on public datasets

9.6.3 Access Controls

  • OpenWebUI: Account-based access
  • HuggingFace: Gated dataset with terms acceptance
  • Processing scripts: Maintainer access only

9.7 Third-Party Services

9.7.1 HuggingFace

  • Purpose: Dataset hosting and distribution
  • Data sent: Processed contributions (post-PII removal)
  • Their terms: https://huggingface.co/terms-of-service

9.7.2 AI Providers (via OpenWebUI)

  • When: During normal chat usage
  • What: Prompts and system messages
  • Control: See provider directory in Settings

9.8 Compliance Notes

9.8.1 GDPR Considerations

  • Legal basis: Consent (explicit opt-in)
  • Data minimization: Only necessary fields collected
  • Purpose limitation: Only for declared research/evaluation
  • Rights: Access, rectification, erasure (with limitations)

9.8.2 Limitations on Deletion

Per our Terms of Service: - Contributions are permanent once published - We can exclude from future releases - We cannot retract from existing public copies - This is clearly disclosed before contribution

9.9 Contact Information

  • Privacy requests: privacy@publicai.company
